# WhoSpy (誰是臥底連線版) - 專案架構說明書

## 1. 專案目標與核心價值

本專案旨在解決傳統「誰是臥底」遊戲在人數多時的痛點，透過網頁連線與自動化系統提升遊戲流暢度。

### 解決痛點對照表

| 原始痛點         | 系統解決方案                                           |
| :--------------- | :----------------------------------------------------- |
| 輪流發言時間過長 | **同時輸入**：所有玩家可同時輸入提示文字。             |
| 難以記住他人提示 | **文字記錄**：系統列出所有回合的提示清單，供隨時查閱。 |
| 環境吵雜聽不清   | **數位顯示**：提示以文字形式呈現於手機螢幕。           |
| 投票難辨識與統計 | **系統投票**：數位匿名投票，系統自動統計並揭露結果。   |
| 淘汰者等待時間長 | **流程加速**：透過計時器與同步操作縮短單局時間。       |

---

## 2. 技術棧 (Tech Stack)

- **Frontend**: React (Vite) - 負責 UI 渲染、一機多人循環介面。
- **Backend**: Node.js + Colyseus - 負責房間邏輯、計時器、狀態同步 (State Sync)。
- **Communication**: WebSocket - 確保文字提示與投票指令的順序性與準確性。
- **Architecture**: Monorepo (pnpm/npm workspaces) - 共享前後端 TypeScript 型別 。

---

## 3. 核心數據結構 (Schema)

定義於 `packages/server/src/schema/State.ts`，由伺服器端維護並即時同步至前端。

- **RoomState**:
  - `phase`: `WAITING` | `SHOW_WORD` | `INPUT_HINT` | `VOTING` | `RESULT`
  - `players`: Map<string, Player> (Key 為 SessionID)
  - `hints`: Array<HintObject> (紀錄所有回合的提示文字與對應玩家)
  - `settings`: 包含題目、臥底人數、時限等配置。

- **Player**:
  - `id`: 玩家唯一辨識碼。
  - `name`: 玩家名稱。
  - `role`: `CIVILIAN` | `SPY` (僅伺服器端知曉，前端依權限顯示)。
  - `isAlive`: 是否存活。
  - `deviceGroup`: 標記該玩家所屬的物理裝置 (用於一機多人模式)。

---

## 4. 遊戲流程狀態機 (Game Phases)

1. **WAITING (等待中)**:
   - 主持人設置遊戲參數。
   - 玩家透過房號加入，並設定該裝置的「參與人數」。
2. **SHOW_WORD (查看題目)**:
   - 系統分配角色與詞彙。
   - **一機多人處理**：前端依序顯示「點選查看」遮罩，確保多人共用時不被偷看 。
3. **INPUT_HINT (輸入提示)**:
   - 全體存活玩家同時輸入文字。
   - **一機多人處理**：該裝置會循環彈出輸入框，直到所有登記玩家完成。
4. **VOTING (投票階段)**:
   - 顯示當前與過往所有回合的提示列表。
   - 玩家選擇投票對象。
5. **RESULT (結算揭露)**:
   - 公佈最高票者及其身分，並更新 `isAlive` 狀態。
   - 由主持人決定進行「下一輪提示」或「再次投票」。

---

## 5. 一機多人 (Multi-user) 特殊邏輯

這是本專案的核心特色，需嚴格遵守以下實作準則：

- **連線與玩家分離**：一個 `Client ID` (物理手機) 可對應多個 `Player` 物件。
- **循環 UI (Sequential UI)**：
  - 當進入 `SHOW_WORD` 或 `INPUT_HINT` 階段，前端需根據該裝置關聯的玩家清單，執行 `for` 循環操作。
  - 例如：玩家 A 輸入完畢並點擊確認後，介面立即切換為玩家 B 的輸入框，直至該裝置全數完成才向伺服器發送 `commit` 。
- **投票統計**：伺服器需檢核該裝置發出的投票數量是否與登記人數相符。

---

## 6. 開發規範與安全建議

- **權限控制**：所有身分判斷 (誰是臥底) 必須保留在伺服器端，僅發送該連線玩家「被授權看到」的資訊。
- **計時器**：伺服器端為主，時間到自動強制切換 Phase。
- **UI 響應**：前端需精準監聽 `Room.state.phase` 的變化，並自動切換對應組件。
